# Watchlist / 内置信源与关注名单

Use this reference when choosing built-in information sources, recurring search methods, or X/Twitter accounts for Silicon Valley Daily.

Do not treat this list as final truth. Use it as a discovery and routing layer. Verify facts through primary sources or reputable reporting before including them in a brief.

## Daily Source Stack

Primary sources:

- Company newsrooms and blogs: OpenAI, Anthropic, Google DeepMind, Google, Meta, Apple, Microsoft, Amazon/AWS, Nvidia, AMD, Databricks, Snowflake, Salesforce, Stripe, Hugging Face, Perplexity, Cursor/Anysphere, Scale AI, Safe Superintelligence, Mistral AI, Cohere.
- Developer and product channels: API docs, changelogs, model cards, status pages, GitHub releases, research papers, technical reports.
- Financial and legal sources: SEC filings, investor relations pages, court filings, regulator pages, state business records, acquisition announcements.
- Startup and VC sources: YC, a16z, Sequoia, Benchmark, Greylock, Founders Fund, Lightspeed, Menlo Ventures, General Catalyst, Accel, Kleiner Perkins, Felicis, Index Ventures, SV Angel.

Reputable context sources:

- Reuters Technology, Bloomberg Technology, Wall Street Journal Tech, Financial Times Tech, CNBC Tech.
- TechCrunch, The Verge, Wired, The Information, Axios Technology, Semafor, Fortune Term Sheet.
- Platformer, Stratechery, Import AI, Latent Space, The Batch, Newcomer, Big Technology.
- San Francisco Chronicle, San Francisco Business Times, Stanford News, Berkeley News when local ecosystem context matters.

Community and weak-signal sources:

- Hacker News, Product Hunt, GitHub Trending, Reddit communities, X/Twitter, LinkedIn, job postings, app store rankings, vendor forums.
- Use these to discover leads, not to confirm facts.

## Daily Research Method

For a panorama brief:

1. Run broad date-bound searches for "Silicon Valley tech today", "Bay Area startup funding", "AI model release", "tech acquisition", and "AI regulation".
2. Run category-specific searches for each focus area.
3. Check the source stack above for primary announcements.
4. Scan X/Twitter accounts below for leads and expert reactions.
5. Confirm each candidate with a primary source or reputable context source.
6. De-duplicate by event and rank by the rubric in `source-strategy.md`.

For a focused brief:

1. Search the selected focus category first.
2. Check official sources for the most relevant companies, VCs, labs, or regulators.
3. Scan 8 to 15 X/Twitter accounts from the matching category.
4. Add out-of-focus events only when they are major enough to change the day.

For recurring use:

- Rotate sources so the brief does not become one publication's worldview.
- Keep a "seen yesterday" memory if the host platform supports memory, and suppress repeated stories unless there is a new fact.
- Periodically refresh this watchlist because X handles, roles, and influence change.

## Account Handling Rules

- Verify the account exists and the role still matches before using it in a brief.
- Prefer original posts over screenshots, quote-tweets, or summaries.
- Treat posts from CEOs, founders, researchers, VCs, and reporters as leads unless they directly announce their own company, fund, paper, filing, or product.
- If a post is central to the brief, link the post and add a primary or reputable context source when possible.
- Avoid including X-only rumors unless the output explicitly labels them as unverified signals.
